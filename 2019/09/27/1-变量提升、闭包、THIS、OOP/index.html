1、Node

  1. Node安装
    1. 下载安装https://nodejs.org/zh-cn
    2. 配置环境变量(添加node安装目录)
    3. 验证
      1. node -v
      2. node –version
      3. npm -v
  2. Node概念
    1. 基于v8引擎渲染js的工具
    2. 执行js代码
      1. cmd node xxx.js
      2. WB中右键run
      3. REPL(输入求值输出循环):cmd 输入NODE回车
  3. NPM模块管理(安装node自带)
    1. 官网下载(速度慢)
      • npm install xxx 下载到当前目录
      • npm install xxx -g 下载到全局
      • npm uninstall xxx
      • npm uninstall xxx -g
      • npm view jQuery>xxx.xxx 输出模块版本信息到文件
    2. nrm切换下载源
      • npm install nrm -g
      • nrm ls 查看可用源
      • nrm use xxx 使用某个源
      • 切完源还是基于npm安装操作
    3. 使用yarn下载(速度最快，不能安装全局)
      • npm install yarn -g
      • yarn add xxx
      • yarn add xxx@x.x.x 下载固定版本
      • yarn remove xxx

2、gitHub

  1. 特点
    1. 分布式代码管理系统
    2. 记录代码所有版本
    3. 方便回退某个版本
    4. 检测代码冲突，合并代码
  2. 创建仓库
    • 配置基础信息
      • $ git config -l
      • $ git config –global user.name xxx
      • $ git config –global user.email xxx
    • $ git init 创建空仓库 会生成.git隐藏文件
    • 创建.gitignore文件，储存git提交时忽略的内容
  3. 工作原理和操作
    • 三个区域
      • 工作区：编辑代码
      • 暂存区：存储临时代码
      • 历史区：生成代码版本
    • 本地操作
      • $ git add . 或 git add -A 工作区到暂存区
      • $ git commit -m’备注信息’ 暂存区到历史区
      • $ git log 查看版本信息
      • $ git diff 比较 工作区 暂存区
      • $ git diff master 比较 工作区 历史区
      • $ git diff –cached 比较 暂存区 历史区
    • git和gitHub同步
      • $ git remote -v 查看关联信息
      • $ git remote add xxx [远程仓库地址] 建立关联
      • $ git remote remove xxx 移除关联
      • $ git pull xxx(关联名) xxx(远程仓库分支) 拉取信息
      • $ git push xxx(关联名) xxx(远程仓库分支) 推送
  4. 真实项目开发流程
    1. leader建立远程仓库、
    2. git init
    3. git clone [远程仓库地址]
    4. git add .
    5. git commit -m
    6. git pull
    7. git push

###3、变量提升

  1. 概念

    1. 作用域形成后先进行变量提升（函数是先进行形参赋值）

    2. var提前声明，function提前定义

    3. 全局作用域下声明一个变量，相当于给window设置一个属性，变量的值就是属性值，修改一方的值另一方会改变，存在映射关系

    5. 只对等号左边进行变量提升

    6. 当前作用域下，不管条件成立否，都进行变量提升，且var和function都只声明（老版本浏览器function会定义），执行到条件成立时形成块级作用域，第一件事是变量提升（条件里面function会先定义）

    7. 名字重复了会重新赋值（不管变量提升还是代码执行）

  2. 作用域链：

    • 如果当前作用域中没有这个私有变量，会向上级作用域查找，一直找到window为止，如果window也没有，则把变量添加到window属性中。
    • 函数的上级作用域只和他在哪里创建的有关

  3. let

    • let不存在变量提升，切断了全局变量和window属性映射机制

    • 代码执行前，浏览器会做重复性检测（词法分析）自上而下查找当前作用域下所有变量，let不允许在同一个作用域下重复定义变量

  4. js堆栈内存释放

    • 堆内存释放：让所有引用堆内存地址的变量赋值为null即可
    • 栈内存释放：函数执行完会释放所形成的作用域，但当函数的栈内存中有内存被以外的某个变量占用了，此时栈内存不会释放，全局栈内存只会在浏览器关闭时释放

4、arguments对象

  1. 概念：arguments是函数的一个内置属性，他的值是一个类数组。
  2. arguments对象的长度是由实参个数决定（即使这个值传递的是undefined）。
  3. 对于arguments和值都存在的情况下，两者值是同步的，但是针对其中一个无值的情况下，对于此无值的情形值不会得以同步。

4、闭包

  1. 概念：形成一个不销毁的私有作用域，保护里面的私有变量不受外界干扰。

  1. 异步和同步
    • 同步：一件一件的做，当前这件事没完成，下一个任务不能处理。
    • 异步：一件一件的做，当前这件事没完成，继续下一个任务。

5、this

  1. 规律
    • 事件绑定，this是当前操作的对象
    • 普通函数执行，函数前有.，前面是谁this就是谁，没有点this是window
    • 自执行函数中的this是window
    • 构造函数执行，this指向实例

###6、OOP

  1. 创建对象

    • 基于函数（类）创建实例，会在函数执行时创建一个堆内存，存放对象，函数中只有this和这个对象有关系
    • 基于函数（类）创建实例，函数的返回默认值是对象，不是undefined
    • 如果指定返回值，当返回值是基本类型，还是返回对象，返回值是引用类型会返回引用类型

  2. 单例设计模式：每一个命名空间都是JS中Object这个内置基类的实例，而实例之间是相互独立互不干扰的，所以我们称它为“单例：单独的实例”

  1. 对象属性的检测

    1. instanceof:判断某个实例是否隶属于这个类
    2. in：检测当前对象是否存在某个属性（公、私）
    3. hasOwnProperty:检测当前属性是否为对象的私有属性

7、原型和原型链

  1. 概念
    1. 所有函数自带一个属性：prototype，属性值是一个对象,存储供实例调用的公共方法（如果当前对象没有这个属性，会向他所属类上的prototype上查找，如果也没有，会向所属类上的prototype上的proto上查找…）
    2. 这个对象自带属性constructor,属性值是函数本身
    3. 每个对象自带_ proto _的属性，这个属性指向当前实例所属类的prototype(如果不能确定他是谁的实例，都是Object的实例)
  2. 进阶
    • 可以重定向类的prototype，需要定义constructor
    • 不能重定向内置类的prototype
